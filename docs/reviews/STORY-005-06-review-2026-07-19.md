## Critical Issues (Must Fix Before Merge)

### Issue #1: Invalid Type Annotation in CelebrationRenderer
- **Severity:** 🔴 Critical
- **Location:** `src/ui/celebration_renderer.py:46`
- **Problem:** Uses `Optional[callable]` instead of proper Callable type annotation
- **Impact:** This is a type annotation error that could cause runtime issues and violates type safety conventions
- **Recommended Fix:** Import `Callable` from typing and use proper annotation

**Current (incorrect):**
```python
from typing import Optional, Tuple
# ...
self.on_celebration_complete: Optional[callable] = None
```

**Fixed (correct):**
```python
from typing import Optional, Tuple, Callable
# ...
self.on_celebration_complete: Optional[Callable[[], None]] = None
```

### Issue #2: Missing Test File for CelebrationRenderer
- **Severity:** 🔴 Critical

- **Problem:** Story requires 40+ unit tests; only 25 tests exist (14 sparkle + 11 planet_bloom). Missing tests for CelebrationRenderer component.

### Issue #2: No Accessibility/Skip Option
- **Severity:** 🟡 Medium
- **Location:** `src/screens/planet_results.py`, `src/ui/celebration_renderer.py`
- **Problem:** Story notes explicitly mention "Animation Skip: Provide option to skip celebration" and "Accessibility: Reduce sparkle intensity for sensitivity" - neither implemented
- **Impact:** Students with visual sensitivities cannot skip or reduce effects; rapid replays must wait through full 3.5s animation
- **Recommended Fix:**
  - Add `skip_celebration()` method to CelebrationRenderer
  - Add input handling in planet_results.py to allow skip (any key click)
  - Add animation intensity check (from player_preferences.py) to reduce particle count

### Issue #3: Rocket Integration Incomplete
- **Severity:** 🟡 Medium
- **Location:** `src/ui/celebration_renderer.py:83, 117, 137, 158, 178`
- **Problem:** Code calls `rocket.start_celebration()`, `rocket.update_celebration()`, `rocket.stop_celebration()`, `rocket.render_celebration()` but STORY-005-02 (Rocket Animations) may not have these methods implemented
- **Impact:** Runtime AttributeError if rocket methods don't exist; celebration may crash
- **Recommended Fix:**
  - Verify RocketSprite/RocketAnimator has these methods OR
  - Add defensive checks before calling rocket methods OR
  - Document dependency on STORY-005-02 rocket implementation

### Issue #4: No Object Pooling for Particles
- **Severity:** 🟡 Medium
- **Location:** `src/ui/sparkle_effect.py:110-122`
- **Problem:** Story implementation tips say "Pre-allocate particle objects (object pooling)" but new Particle objects created on every emit
- **Impact:** Potential memory churn and GC pressure during extended celebrations; may violate "<5MB RAM" target over time
- **Recommended Fix:** Implement simple object pool:
  - Pre-allocate pool of 100 particles
  - Reuse dead particles instead of creating new ones
  - Track active/inactive particles separately

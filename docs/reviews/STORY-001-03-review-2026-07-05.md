## Critical Issues (Must Fix Before Merge)

### Issue #1: Auto-Advance Timer Logic Bug
- **Severity:** 🔴 Critical
- **Location:** `src/components/feedback_controller.py:168-175`
- **Problem:** The `_complete_feedback()` method checks if the auto-advance timer has expired, but this check happens AFTER the feedback is already complete. The auto-advance callback is triggered inside `_complete_feedback()`, but the timer comparison `time.time() >= self.auto_advance_timer` may fail because the timer was set to `feedback_start_time + success_display_duration`, and by the time `_complete_feedback()` is called, the condition might not be met correctly.
- **Impact:** Auto-advance may not trigger reliably, causing the game to get stuck on the success feedback screen.
- **Recommended Fix:** Separate the auto-advance trigger from feedback completion. The auto-advance should be scheduled independently:

```python
# Current (problematic):
def _complete_feedback(self):
    self.animation_active = False
    self.state = FeedbackState.IDLE
    # ... 
    # Trigger auto-advance for success
    if self.auto_advance_timer and time.time() >= self.auto_advance_timer:
        if self.on_auto_advance:
            self.on_auto_advance()

# Fixed:
def _show_success(self) -> bool:
    # ... existing code ...
    if self.config.auto_advance_on_success:
        # Schedule auto-advance separately
        def schedule_advance():
            time.sleep(self.config.success_display_duration)
            if self.on_auto_advance:
                self.on_auto_advance()
        threading.Thread(target=schedule_advance, daemon=True).start()
    return True
```

**Alternative simpler fix:** Check for auto-advance in the `update()` method when the duration has elapsed:

```python
def update(self, current_time: float):
    if self.state == FeedbackState.SHOWING_SUCCESS:
        if elapsed >= self.config.success_display_duration and not self._advanced:
```

### Issue #2: Missing Audio Asset Files
- **Severity:** 🔴 Critical
- **Location:** `assets/audio/` directory
- **Problem:** The story file specifies creating `assets/audio/success_chime.ogg` and `assets/audio/retry_tone.ogg`, but these files were NOT created. The implementation uses TTS as a fallback, but the acceptance criteria specifically require "success chime (ascending C-E-G chord)" and "soft descending tone" which are specific sound effects, not TTS speech.
- **Impact:** The audio feedback does not match the design specification. TTS saying "Great job!" is different from a success chime sound.
- **Recommended Fix:** 
  1. Create the actual audio files as specified in the story
  2. Update `audio_system.py` to play these files instead of using TTS for feedback
  3. Keep TTS as a fallback for when audio files are unavailable

```python
# In _play_success_sound():
def _play_success_sound(self):
    try:
        # Play the actual chime file
        self.audio_system.play_sfx('success_chime')  # New method for SFX
    except Exception:
        # Fallback to TTS
        self.audio_system.speak(self.config.success_message)
```

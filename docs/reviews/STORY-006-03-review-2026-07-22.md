## Critical Issues (Must Fix Before Merge)

### Issue #1: No Audio System Integration

- **Severity:** 🔴 Critical
- **Location:** `src/components/audio_system.py`, `src/components/caption_manager.py`
- **Problem:** CaptionManager is not integrated with AudioSystem. When TTS speaks or SFX plays, no captions are automatically triggered.
- **Impact:** Feature is unusable - captions never appear during gameplay despite both systems being implemented.
- **Recommended Fix:**
  1. Add `caption_manager` parameter to AudioSystem.__init__()
  2. In `speak()` method, trigger caption when TTS starts:
     ```python
     def speak(self, text: str, on_complete: Optional[Callable] = None) -> bool:
         # Trigger caption if available
         if self.caption_manager:
             self.caption_manager.show_caption(text, speaker="TTS")
         # ... rest of speak implementation
     ```
  3. In `play_sfx()` method, trigger SFX description caption:
     ```python
     def play_sfx(self, sfx_name: str, volume: float = 1.0) -> bool:
         # Trigger SFX caption if available
         if self.caption_manager and sfx_name in SFX_CAPTION_MAP:
             self.caption_manager.show_caption_by_id(
                 "sfx_descriptions", sfx_name, is_sfx=True
             )
         # ... rest of play_sfx implementation
     ```
  4. Create a mapping of SFX names to caption keys
  5. Update game screens to pass CaptionManager to AudioSystem

### Issue #2: No Game Screen Integration

- **Severity:** 🔴 Critical
- **Location:** All game screens (spelling_challenge.py, main_menu.py, etc.)
- **Problem:** No game screen initializes or uses CaptionManager. Captions are completely disconnected from gameplay.
- **Impact:** Even with audio integration fixed, captions won't display because no screen manages the CaptionManager lifecycle.
- **Recommended Fix:**
  1. Add CaptionManager initialization to Game class or main screen:
     ```python
     # In src/game.py or main screen
     from src.components.caption_manager import CaptionManager
     from src.components.caption_settings import CaptionSettingsManager
     
     def __init__(self):
         # ... existing init ...
         
         # Initialize caption system
         settings_mgr = CaptionSettingsManager()
         self.caption_settings = settings_mgr.get_settings()
         self.caption_display = CaptionDisplay(self.screen, self.caption_settings)
         self.caption_manager = CaptionManager(self.caption_display)
     ```
  2. Add `update()` call in game loop:
     ```python
     def update(self, delta_time: float):
         # ... existing updates ...
         self.caption_manager.update(delta_time)
     ```
  3. Integrate with Captain Cosmos by adding caption-triggering calls:
     ```python
     # When Captain Cosmos speaks
     self.audio_system.speak(line, on_complete=callback)
     self.caption_manager.show_caption_by_id(
         "captain_cosmos", key, speaker="Captain Cosmos"
     )
     ```

## Architecture Concerns

The current implementation follows a **component-based architecture** that is theoretically sound but **practically disconnected**:

```
CaptionManager → CaptionDisplay → Pygame Surface
       ↓
   CaptionSettings (persistent config)
```

However, there's no bridge to:
- `AudioSystem` (no caption triggers on speak/SFX)
- Game screens (no CaptionManager lifecycle)
- Parent Dashboard (no settings UI)

This suggests the story was implemented **in isolation** without considering how it fits into the larger system roadmap.

## Critical Issues (Must Fix Before Merge)

### Issue #1: Duplicate Code - RocketAnimator in Two Files

- **Severity:** 🔴 Critical (Code Quality / Architecture)
- **Location:** `src/screens/planet_transition.py` (lines 25-105) AND `src/ui/animations.py` (lines 21-101)
- **Problem:** The `RocketAnimator` class is implemented **twice** - once in `planet_transition.py` as a module-level class, and again in `animations.py` as a shared utility class. These classes are identical in functionality.
- **Impact:** 
  - Code duplication violates DRY principle
  - Future changes must be made in two places
  - Confusion for developers about which import to use
  - Increased bundle size and maintenance burden
- **Recommended Fix:**

  Remove the `RocketAnimator` class from `planet_transition.py` and import it exclusively from `animations.py`:

  ```python
  # Remove these lines from planet_transition.py:
  # @dataclass  
  # class RocketAnimator:
  #     ... entire class definition ...

  # Add this import at the top:
  from src.ui.animations import RocketAnimator
  ```

### Issue #2: Redundant StarField in planet_transition.py

- **Severity:** 🟠 High (Code Quality)
- **Location:** `src/screens/planet_transition.py` (lines 142-201)
- **Problem:** The `StarField` class is defined in `planet_transition.py` but should be in `animations.py` alongside other animation components. This creates the same duplication problem as Issue #1.
- **Impact:** Same as Issue #1 - maintenance burden and confusion
- **Recommended Fix:**

  Move `StarField` to `animations.py` and import it in `planet_transition.py`:

  ```python
  # Remove from planet_transition.py:
  # class StarField: ...

  # Add import:
  from src.ui.animations import StarField
  ```

## Medium Priority Issues (Should Fix Before Merge)

### Issue #1: Incomplete Progress Bar Animation Integration

- **Severity:** 🟡 Medium
- **Location:** `src/ui/animations.py` (lines 183-220)
- **Problem:** `ProgressBarAnimator` exists in `animations.py` but is NOT used by `PlanetTransitionScreen`. The transition screen draws the progress bar manually in `_draw_progress_bar()` instead of using the reusable animator.
- **Impact:** Missed opportunity for reusable animation component; progress bar animation is not smooth
- **Recommended Fix:**

  Integrate `ProgressBarAnimator` into `PlanetTransitionScreen`:

  ```python
  class PlanetTransitionScreen:
      def __init__(self, typography: Typography, audio_system: AudioSystem):
          # ... existing init ...
          from src.ui.animations import ProgressBarAnimator
          self.progress_bar = ProgressBarAnimator()
  
      def update(self, delta_time: float):
          # ... existing update ...
          self.progress_bar.set_target(self.galaxy_progress)
          self.progress_bar.update(delta_time)
  
      def _draw_progress_bar(self, screen):
          bar_width = 400
          bar_height = 20
          bar_x = (self.width - bar_width) // 2
          bar_y = self.height - 80
          
          # Use the animator instead of manual drawing
          self.progress_bar.draw(screen, bar_x, bar_y, bar_width, bar_height)
  ```

### Issue #2: Hardcoded "12 Planets" Assumption

- **Severity:** 🟡 Medium
- **Location:** `src/components/progress_tracker.py` (lines 44-47, 338-342)
- **Problem:** The `GalaxyProgress` dataclass defaults to `total_planets: int = 12`, but the story notes state "Galaxy progress assumes 12 planets; should be configurable via word list metadata." This hardcoded value could cause issues if the word list changes.
- **Impact:** Inflexible architecture; will require code changes to support different galaxy sizes
- **Recommended Fix:**

  Remove the default and require explicit configuration:

  ```python
  @dataclass
  class GalaxyProgress:
      """Galaxy/solar system progress tracking."""
      total_planets: int = 0  # Must be explicitly set
      completed_planets: int = 0
      current_planet_number: int = 1
      unlocked_planets: int = 0
  ```

  And update callers to configure `total_planets` from word list metadata:
  
  ```python
  # In game initialization:
  tracker.set_total_planets(word_list_metadata.total_planets)
  ```

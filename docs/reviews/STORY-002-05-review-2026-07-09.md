## Medium Priority Issues (Should Fix)

### Issue #1: Race Condition Risk in Mastery Detection
- **Severity:** 🟡 Medium
- **Location:** `src/components/progress_tracker.py:289-297`
- **Problem:** The code captures `would_be_mastered` state BEFORE calling `session_tracker.complete_word()`, which is correct, but relies on accessing private attributes (`_word_first_attempt_correct`, `_word_hints_used`) from `SessionTracker`. This creates a tight coupling and potential fragility if `SessionTracker` internal structure changes.
- **Impact:** Future refactoring of `SessionTracker` could silently break mastery detection
- **Recommended Fix:** 
  - Add a public method to `SessionTracker` like `is_word_mastered()` that returns the mastery status
  - Use that method instead of accessing private attributes directly
- **Example:**
  ```python
  # Current (fragile - accessing private attrs):
  would_be_mastered = (
      is_correct and 
      session_tracker._word_first_attempt_correct is True and
      session_tracker._word_hints_used == 0
  )
  
  # Recommended (robust - public API):
  would_be_mastered = session_tracker.is_current_word_mastered() and is_correct
  ```

### Issue #2: Empty State Message Text Mismatch
- **Severity:** 🟡 Medium
- **Location:** `src/components/progress_tracker.py:354` and `src/ui/practice_list.py:66-73`
- **Problem:** Story acceptance criteria specifies empty state message should be "Great job! No words need practice" but implementation uses "Great job! All words are mastered!" 
- **Impact:** Minor deviation from specified UX requirement
- **Recommended Fix:** Update `get_practice_list_empty_message()` to return exact text from story:
  ```python
  def get_practice_list_empty_message(self) -> str:
      return "Great job! No words need practice"
  ```
  
  And update student view in `practice_list.py`:
  ```python
  # Change from:
  message = "Amazing! All words are mastered!"
  # To:
  message = "Great job! Keep it up!"  # As specified in story
  ```

## Critical Issues (Must Fix Before Merge)

### Issue #1: Missing Path Traversal Validation
- **Severity:** 🔴 Critical
- **Location:** `src/words/csv_importer.py:285-295`, `src/ui/csv_import_dialog.py:180-190`
- **Problem:** User-uploaded CSV files are read without validating the file path, potentially allowing path traversal attacks (e.g., `../../etc/passwd`)
- **Impact:** An attacker could read arbitrary files on the system accessible to the application process
- **Recommended Fix:** Add path validation to ensure the file is within allowed directories

**Example Fix:**
```python
import os

def is_safe_path(filepath: str, allowed_dir: str) -> bool:
    """Check if filepath is within allowed directory."""
    abs_path = os.path.abspath(filepath)
    abs_allowed = os.path.abspath(allowed_dir)
    return abs_path.startswith(abs_allowed + os.sep) or abs_path == abs_allowed

# In CSVImporter.parse_file():
if not is_safe_path(self.filepath, os.getcwd()):  # Or a specific upload directory
    return ImportResult(
        skipped=[(0, "Invalid file path")], 
        total_processed=0
    )
```

## Medium Priority Issues (Should Fix)

### Issue #1: Error Messages Reveal Internal Details
- **Severity:** 🟡 Medium
- **Location:** `src/words/csv_importer.py:290-296`
- **Problem:** Error messages include detailed exception text that could reveal system information
- **Impact:** Information disclosure to users about file system structure
- **Recommended Fix:** Sanitize error messages

**Example Fix:**
```python
except FileNotFoundError:
    return ImportResult(
        skipped=[(0, "File not found")],
        total_processed=0
    )
except Exception as e:
    # Log detailed error internally, show generic message to user
    logger.error(f"CSV parse error: {e}")
    return ImportResult(
        skipped=[(0, "Could not read file")],
        total_processed=0
    )
```

### Issue #2: No File Size Limit
- **Severity:** 🟡 Medium
- **Location:** `src/words/csv_importer.py:285-295`
- **Problem:** No maximum file size check, potentially allowing DoS via large file uploads
- **Impact:** Application could be crashed by uploading a multi-gigabyte CSV file
- **Recommended Fix:** Add file size validation

**Example Fix:**
```python
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB

def parse_file(self):
    # Check file size
    if self.filepath:
        file_size = os.path.getsize(self.filepath)
        if file_size > MAX_FILE_SIZE:
            return ImportResult(
                skipped=[(0, f"File too large (max {MAX_FILE_SIZE // 1024 // 1024}MB)")],
                total_processed=0
            )
    # ... continue parsing
```

### Issue #3: Missing chardet Library Dependency
- **Severity:** 🟡 Medium
- **Location:** `src/words/csv_importer.py:18-110`
- **Problem:** The story file mentions `chardet` library for encoding detection, but the implementation uses a fallback heuristic approach. The dependency is not listed in requirements.
- **Impact:** Implementation works but may not detect all encodings correctly
- **Recommended Fix:** Either add `chardet` to requirements and use it, or document the heuristic approach as MVP

### Issue #4: Duplicate ID Generation Risk
- **Severity:** 🟡 Medium
- **Location:** `src/ui/csv_import_dialog.py:292-293`
- **Problem:** Word IDs are generated as `csv_{N}` where N is based on current word count. Concurrent imports could generate duplicate IDs.
- **Impact:** Potential data corruption or overwrites
- **Recommended Fix:** Use UUID or timestamp-based IDs

**Example Fix:**
```python
import uuid

# Generate unique ID
word_id = f"csv_{uuid.uuid4().hex[:8]}"
```

### Issue #2: Particle Memory Leak
- **Severity:** 🟡 Medium
- **Location:** `src/ui/badge_notification.py:119-140`
- **Problem:** BadgeNotification creates 50 confetti particles in `__init__()` but never cleans them up. If multiple badges unlock in one session, particles accumulate in memory.
- **Impact:** Memory leak that could cause performance degradation over long sessions.
- **Recommended Fix:** Clear particles when notification completes:
  ```python
  def is_complete(self) -> bool:
      """Check if notification is complete."""
      if self.elapsed >= self.total_duration:
          self.particles.clear()  # Add this line
          return True
      return False
  ```

### Issue #6: No Performance Validation Tests
- **Severity:** 🟡 Medium
- **Location:** `tests/test_badge_system.py`
- **Problem:** Acceptance criterion requires badge unlock check completes in <50ms, but no tests verify this.
- **Impact:** Cannot guarantee performance requirement is met.
- **Recommended Fix:** Add performance test:
  ```python
  import time
  
  def test_badge_unlock_performance():
      """Verify badge unlock check completes in <50ms."""
      manager = create_badge_manager()
      manager.start_session()
      
      start = time.time()
      for _ in range(100):
          manager.on_word_started()
          manager.on_word_completed(
              attempts=1, hints_used=0,
              is_first_attempt_correct=True,
              completion_time=20.0
          )
      elapsed = (time.time() - start) * 1000  # Convert to ms
      
      # 100 operations should complete in <500ms (5ms each on average)
      assert elapsed < 500, f"Badge unlock check took {elapsed}ms, expected <500ms"
  ```

### Issue #2: No Game Screen Integration

- **Severity:** 🔴 Critical
- **Location:** `src/game.py`, all game screens
- **Problem:** No game screen initializes CaptionManager, CaptionDisplay, or CaptionSettings. There's no parent object to coordinate the caption system lifecycle.
- **Impact:** Even with audio integration fixed, captions won't display because:
  - No CaptionManager instance exists
  - No CaptionDisplay is created with a screen reference
  - No update() call wakes up the caption system each frame
  - No caption settings are loaded or applied
- **Recommended Fix:**
  
  1. Initialize caption system in Game class:
     ```python
     # src/game.py
     from src.components.caption_manager import CaptionManager
     from src.components.caption_settings import CaptionSettingsManager, CaptionSettings
     from src.ui.caption_display import CaptionDisplay
     
     class Game:
         def __init__(self):
             # ... existing initialization ...
             
             # Initialize caption system
             try:
                 caption_settings_mgr = CaptionSettingsManager()
                 self.caption_settings = caption_settings_mgr.get_settings()
                 self.caption_display = CaptionDisplay(self.screen, self.caption_settings)
                 self.caption_manager = CaptionManager(self.caption_display)
                 
                 # Pass caption_manager to AudioSystem
                 self.audio_system = AudioSystem(caption_manager=self.caption_manager)
                 
                 logger.info("Caption system initialized")
             except Exception as e:
                 logger.error(f"Failed to initialize caption system: {e}")
                 self.caption_manager = None
     ```
  
  2. Add caption update to game loop:
     ```python
     def update(self, delta_time: float):
         # ... existing updates ...
         
         # Update captions (call every frame)
         if self.caption_manager:
             self.caption_manager.update(delta_time)
     ```
  
  3. Integrate with Captain Cosmos voice lines:
     ```python
     # Example: In a screen where Captain Cosmos speaks
     def show_captain_message(self, message_key: str):
         # Get the caption text
         caption_text = self.caption_manager.get_caption(
             "captain_cosmos", message_key
         )
         
         # Speak the message
         self.audio_system.speak(caption_text)
         
         # Show caption (will also be triggered by audio system)
         self.caption_manager.show_caption(
             text=caption_text,
             speaker="Captain Cosmos",
             caption_id=f"captain_{message_key}"
         )
     ```

## Architecture Analysis

### Current Component Structure
```
CaptionSettingsManager (persistence)
        ↓
  CaptionSettings (configuration)
        ↓
CaptionDisplay (rendering) ← CaptionManager (logic)
```

### Gap: Missing Bridges
```
AudioSystem ──X── CaptionManager  ❌ (should connect)
   Game ──X── CaptionManager       ❌ (should manage lifecycle)
ParentDashboard ──X──CaptionSettingsPanel  ❌ (missing UI)
```

### Comment on Architecture
The implementation follows a **classical MVC-inspired pattern** that is sound in isolation:
- **Model**: CaptionSettings (data + persistence)
- **View**: CaptionDisplay (rendering)
- **Controller**: CaptionManager (logic + orchestration)

However, the **integration layer is completely missing**. The component trio exists without:
1. A bridge to AudioSystem (to trigger captions on audio events)
2. A parent container in the game loop (to manage lifecycle)
3. A settings UI (to expose controls to users)

This suggests the story was implemented **in functional isolation** rather than as an integrated feature.

## Medium Priority Issues (Should Fix)

### Issue #1: Missing On-Screen Keyboard Component
- **Severity:** 🟡 Medium
- **Location:** `src/ui/keyboard.py` (file does not exist)
- **Problem:** The story technical implementation specifies creating `src/ui/keyboard.py` for the on-screen keyboard component, but this file was not created. The story requirements state: "Support both physical keyboard and on-screen keyboard (for touch devices)" and "On-screen keyboard needed for tablet/touch deployments."
- **Impact:** The application will not work on touch devices/tablets as intended. The `InputHandler` has `handle_virtual_key()` method ready, but there's no UI component to provide the virtual keyboard interface.
- **Recommended Fix:** Create `src/ui/keyboard.py` with an on-screen keyboard component that:
  - Renders a QWERTY keyboard layout
  - Handles touch/click events to send characters to `InputHandler.handle_virtual_key()`
  - Includes special keys: BACKSPACE, ENTER/SUBMIT
  - Follows the UI styling from `typography.py`
  - Is responsive for different screen sizes

### Issue #2: Unused Import in spelling_challenge.py
- **Severity:** 🟡 Medium
- **Location:** `src/screens/spelling_challenge.py:10`
- **Problem:** The import `from src.utils.validators import InputValidator, AnswerValidator` includes `InputValidator` which is only used in one place (`_check_answer` method), and that method appears to be legacy code that's not actively used in the new validation flow.
- **Impact:** Minor code clutter, potential confusion for future developers.
- **Recommended Fix:** 
  - Either remove the unused `InputValidator` import if `_check_answer` is not needed
  - Or refactor `_check_answer` to use `AnswerValidator` consistently

## Low Priority / Suggestions (Nice to Have)

### Suggestion #1: Add Visual Feedback for Invalid Input
- **Severity:** 🟢 Low / 💡 Suggestion
- **Location:** `src/screens/spelling_challenge.py`
- **Suggestion:** The story mentions "Invalid character pressed → No change, subtle shake animation on input field" but there's no visual feedback mechanism implemented. The `_on_invalid_input` callback exists but isn't connected to any visual feedback.
- **Benefit:** Better UX for students - they'll know why their key press didn't register.

### Suggestion #2: Performance Warning Threshold
- **Severity:** 🟢 Low
- **Location:** `src/screens/spelling_challenge.py:128`
- **Suggestion:** The warning threshold is 200ms, but the story's non-functional requirement specifies "Letter appears on screen within 50ms of key press." Consider adding a separate performance check for key press response time.
- **Benefit:** Ensures the 50ms performance requirement is met.

## Critical Issues (Must Fix Before Merge)

### Issue #1: Hint Letters Not Displayed in UI
- **Severity:** 🔴 Critical
- **Location:** `src/screens/spelling_challenge.py:265-278`
- **Problem:** The `_update_word_display_with_hints()` method builds a `full_display` list but never uses it to update the actual UI. The method ends without applying the revealed letters to the input display.
- **Impact:** Students will see hint messages saying "The 1st letter is 'H'" but the letter will NOT appear in the word display, making hints confusing and ineffective.
- **Recommended Fix:** Complete the method to actually update the input display with revealed letters. The method should call a method on `input_display` to show revealed letters.

**Example Fix:**
```python
def _update_word_display_with_hints(self):
    """Update the word display to show revealed hint letters."""
    if not self.hint_manager or not self.input_display:
        return
    
    # Get the full word with revealed letters
    revealed_pattern = self.hint_manager.get_revealed_pattern()
    revealed_indices = self.hint_manager.get_revealed_indices()
    
    # Build the full display list
    full_display = []
    for i, letter in enumerate(self.current_word.text if self.current_word else ""):
        if i < len(self.starter_letters):
            full_display.append(('starter', letter))
        elif i in revealed_indices:
            full_display.append(('hint', letter))
        else:
            full_display.append(('hidden', '_'))
    
    # CRITICAL: Actually update the display
    # The input_display needs a method to set the full display state
    # This method doesn't exist yet - may need to be added to InputDisplay
    if hasattr(self.input_display, 'set_full_display'):
        self.input_display.set_full_display(full_display)
    else:
        # Fallback: at minimum, log the issue for debugging
        print(f"Warning: input_display lacks set_full_display method. Display would show: {full_display}")
```

**Additional Work Needed:** The `InputDisplay` component may need a new method `set_full_display()` to accept and render the combined display of starter letters, hint-revealed letters, and hidden letters.

## Medium Priority Issues (Should Fix)

### Issue #2: Incorrect Type Hint for Callback
- **Severity:** 🟡 Medium
- **Location:** `src/ui/hint_display.py:82`
- **Problem:** Uses `callable` (built-in function) instead of `Callable` (typing import) for the callback type hint.
- **Impact:** Type checkers (mypy, pyright) will flag this as an error. Does not affect runtime behavior.
- **Recommended Fix:** Import `Callable` from typing and use it correctly.

**Current Code:**
```python
self.on_help_clicked: Optional[callable] = None
```

**Fixed Code:**
```python
from typing import Optional, List, Tuple, Set, Callable

# ...

self.on_help_clicked: Optional[Callable[[], None]] = None
```

### Issue #3: Code Duplication in Pattern Building
- **Severity:** 🟡 Medium
- **Location:** `src/components/hint_manager.py:134-144` and `src/ui/hint_display.py:97-107`
- **Problem:** Both `HintManager.get_revealed_pattern()` and `HintDisplay._build_revealed_pattern()` implement the same logic for building the revealed letter pattern.
- **Impact:** Maintenance burden - if the pattern format changes, both locations must be updated.
- **Recommended Fix:** Have `HintDisplay` use `HintManager.get_revealed_pattern()` directly instead of duplicating the logic.

**Recommended Refactor:**
```python
# In HintDisplay.show_hint():
def show_hint(self, hint_text: str, revealed_indices: Set[int]):
    self.current_hint_text = hint_text
    self.revealed_indices = revealed_indices.copy()
    # Get pattern from HintManager instead of building it here
    # This requires passing hint_manager reference or getting pattern from hint_data
    self.revealed_pattern = self._build_revealed_pattern()  # Keep as fallback
```

## Critical Issues (Must Fix Before Merge)

### Issue #1: Galaxy Explorer Tracking Logic Defective
- **Severity:** 🔴 Critical
- **Location:** `src/components/sticker_manager.py:284-297`
- **Problem:** The Galaxy Explorer unlock logic is fundamentally broken. The code checks `if self.tracker.completed_solar_systems:` which will be true after ANY planet is completed, but the solar system tracking doesn't actually track which solar system the planet belongs to or count completed solar systems correctly. The condition "Complete all planets in a solar system (3 planets)" is not implemented.
- **Impact:** Sticker will unlock incorrectly (after 1 planet instead of 3 planets in a solar system) or not at all depending on code flow.
- **Recommended Fix:** Implement proper solar system tracking:
  ```python
  # In StickerTracker class:
  solar_system_planets: Dict[str, Set[str]] = field(default_factory=dict)  # {solar_system: {planet_ids}}
  
  # In start_planet():
  def start_planet(self, planet_id: str, solar_system: str):
      if solar_system not in self.tracker.solar_system_planets:
          self.tracker.solar_system_planets[solar_system] = set()
      self.tracker.solar_system_planets[solar_system].add(planet_id)
  
  # In on_planet_completed():
  # Check if all planets in the solar system have been completed
  solar_system_planets = self.tracker.solar_system_planets.get(solar_system, set())
  # Need to track TOTAL planets per solar system (this requires knowing total count)
  # For now, use the commonly-known fact: each solar system has 3 planets
  if len(solar_system_planets) >= SOLAR_SYSTEM_PLANETS_REQUIRED and not self.tracker.galaxy_explorer_unlocked:
      self._unlock_sticker('galaxy_explorer', {"system": solar_system})
  ```

### Issue #2: Time-Based Tracking Bug
- **Severity:** 🔴 Critical
- **Location:** `src/components/sticker_manager.py:245, 305-311`
- **Problem:** The `start_session(hour)` initialization happens in `__init__()` at line 200, which occurs BEFORE the user provides the hour parameter. The hour is passed to `start_session()` but the method sets `_current_planet_start_time` to `time.time()` which is used for Speed Demon tracking, creating a timing bug. Additionally, start_session is called from ProgressTracker line 203, but the logic for storage of early/late games is duplicated between `start_session()` (line 254-260) and the tracker attributes on the StickerTracker class.
- **Impact:** Early Bird and Night Owl stickers may unlock incorrectly, and Speed Demon timing may be inaccurate since `_current_planet_start_time` is set at session start, not planet start.
- **Recommended Fix:** Remove the `_current_planet_start_time = time.time()` from `start_session()` since it should only be set in `start_planet()`. Also, consolidate the time-based tracking so that Early Bird/Night Owl counters are incremented once per day (not once per session):
  ```python
  # Remove this from start_session():
  self._current_planet_start_time = time.time()
  
  # Add to start_planet() instead:
  def start_planet(self, planet_id: str, solar_system: str):
      self._current_planet_start_time = time.time()
      # ... rest of method
  
  # Fix Early Bird/Night Owl to not double-unlock:
      # Check if already unlocked for today
      if hour < 9:
          if not self.tracker.early_bird_unlocked:
              self._unlock_sticker('early_bird', {"time": f"{hour}:00"})
  ```

## Medium Priority Issues (Should Fix)

### Issue #4: Speed Demon Timer Bug
- **Severity:** 🟡 Medium
- **Location:** `src/components/sticker_manager.py:290-295`
- **Problem:** The Speed Demon timer uses `self._current_planet_start_time` which is set in `start_planet()`, but if `start_planet()` is called but the planet is never completed (user quits mid-game), the timer state persists for the next planet. Also, the timer is reset in `start_planet()` at line 277 but that line doesn't exist in the current code.
- **Impact:** Speed Demon sticker may unlock incorrectly if timer isn't reset properly.
- **Recommended Fix:** Ensure timer is always properly reset:
  ```python
  def start_planet(self, planet_id: str, solar_system: str):
      self._current_planet_start_time = time.time()
      # Explicitly reset stale state
      if self._current_planet_start_time is None:
          self._current_planet_start_time = time.time()
  ```

### Issue #5: Galaxy Explorer Partial Progress Not Tracked
- **Severity:** 🟡 Medium
- **Location:** `src/components/sticker_manager.py:299-305`
- **Problem:** The progress for Galaxy Explorer sticker is updated using `len(self.tracker.completed_solar_systems)` but this set is never populated anywhere in the code. The progress display will show "0/3" permanently.
- **Impact:** Users won't see their progress toward Galaxy Explorer sticker.
- **Recommended Fix:** Populate `completed_solar_systems` when a solar system is completed:
  ```python
  # In on_planet_completed():
  if len(self.tracker.solar_system_planets.get(solar_system, set())) >= SOLAR_SYSTEM_PLANETS_REQUIRED:
      self.tracker.completed_solar_systems.add(solar_system)
  
  # Then update progress:
  self.progress['galaxy_explorer'].current = len(self.tracker.completed_solar_systems)
  ```

### Issue #6: Type Annotation Error in Callback
- **Severity:** 🟡 Medium
- **Location:** `src/components/sticker_manager.py:176`
- **Problem:** Line 176 uses `Optional[callable]` which is incorrect. `callable` is a built-in function, not a type. Should use `Optional[Callable]` from typing module.
- **Impact:** Type checkers will fail, incorrect type hints.
- **Recommended Fix:** 
  ```python
  from typing import Callable
  self.on_sticker_unlocked: Optional[Callable[[StickerDefinition], None]] = None
  ```

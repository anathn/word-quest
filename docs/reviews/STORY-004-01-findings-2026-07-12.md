## Critical Issues (Must Fix Before Merge)

### Issue #1: Import Statement Inside Method

- **Severity:** 🔴 Critical
- **Location:** `src/components/streak_tracker.py:46`
- **Lines:** 45-47

**Problem:**
```python
def record_correct_answer(self) -> int:
    self._current_streak += 1
    
    if self._current_streak > self._best_streak:
        self._best_streak = self._current_streak
    
    if self._current_streak == 1:
        import time  # ← Import inside method!
        self._streak_start_time = time.time()
```

**Why It's a Problem:**
- **Performance penalty:** Python re-executes the import statement every time a correct answer is recorded
- **Violates conventions:** All imports should be at module level (PEP 8)
- **Hard to maintain:** Dependencies are not visible at the top of the file
- **Testing issues:** Makes mocking/time dependency harder to manage

**Fix:**

1. Add `import time` at the top of the file (after other imports):

```python
# File: src/components/streak_tracker.py
# Lines 1-8 (add import time here):
"""
Streak Tracker Component (STORY-004-01)
...
"""

from typing import Optional
import time  # ← Add this line here
```

2. Remove `import time` from inside the method (lines 45-47):

```python
def record_correct_answer(self) -> int:
    self._current_streak += 1
    
    if self._current_streak > self._best_streak:
        self._best_streak = self._current_streak
    
    if self._current_streak == 1:
        self._streak_start_time = time.time()  # ← Just use time.time()
    
    return self._current_streak
```

## Medium Priority Issues (Should Fix)

### Issue #3: Random Seed Predictability

- **Severity:** 🟡 Medium
- **Location:** `src/ui/streak_display.py:237`

**Problem:**
```python
def _render_star_particles(self, center_x: int, center_y: int) -> None:
    import math
    import random
    
    # Use streak value as seed - makes positions predictable!
    random.seed(self._current_streak)
    
    radius = 50
    for i, star in enumerate(self._star_surfaces):
        angle = (2 * math.pi * i) / len(self._star_surfaces)
        angle += random.uniform(-0.2, 0.2)  # Same variation every time!
```

**Why It's a Problem:**
- **Not truly random:** Same streak value produces identical particle patterns
- **Unrealistic visuals:** Fire/streaks should look different each time
- **Poor practice:** Shows misunderstanding of random seed fundamentals

**Fix:**

Use time-based seed for natural variation:

```python
def _render_star_particles(self, center_x: int, center_y: int) -> None:
    import math
    import time
    import random
    
    # Use time for natural variation (milliseconds for uniqueness)
    time_seed = int(time.time() * 1000) % 100000
    random.seed(time_seed)
    
    radius = 50
    for i, star in enumerate(self._star_surfaces):
        angle = (2 * math.pi * i) / len(self._star_surfaces)
        angle += random.uniform(-0.2, 0.2)
        
        # Rest of code unchanged...
```

**Alternative:** If you want reproducibility for testing:

```python
# Use a hash that includes both streak AND time
import hashlib
seed_value = int(
    hashlib.md5(f"{self._current_streak}_{time.time()}".encode()).hexdigest()[:8],
    16
)
random.seed(seed_value)
```

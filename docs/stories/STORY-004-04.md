## Overview

This story implements Captain Cosmos, the friendly robot mascot character who guides players through the game with encouraging voice lines, appears during key moments, and provides a personal connection to make the game more engaging.

## User Story

As a 3rd grade student, I want a friendly character to talk to and encourage me so that I feel like I have a companion helping me on my spelling adventure.

## Acceptance Criteria

### Functional Requirements

- [ ] Captain Cosmos sprite with multiple animation states (idle, talking, celebrating, encouraging)
- [ ] Voice lines for different game events:
  - **Tutorial**: Guides first-time players through game mechanics
  - **Correct answer**: Encouraging praise ("Great job!", "You're a spelling star!")
  - **Incorrect answer**: Gentle encouragement ("Try again!", "You've got this!")
  - **Streak milestones**: Celebration ("3 in a row! Awesome!")
  - **Planet completion**: Victory praise ("Planet cleared! Excellent work!")
  - **Help requested**: Hint delivery with encouraging tone
- [ ] Character appears in key moments:
  - Main menu introduction
  - Tutorial sequences
  - Planet completion celebrations
  - When help is requested
  - Badge unlocks
- [ ] Voice lines delivered via Text-to-Speech (TTS) for MVP
- [ ] Character doesn't block gameplay (appears in designated area)
- [ ] Optional: Students can choose Captain's voice (pitch/tempo variations)

### Non-Functional Requirements

- [ ] Performance: Voice line starts within 200ms of trigger
- [ ] Animation: Smooth transitions between states (≤300ms)
- [ ] Audio: Volume at 50% of SFX by default, adjustable
- [ ] Accessibility: All voice lines have closed captions

### UI/UX Requirements

- [ ] Captain position: Bottom-left corner during gameplay (non-intrusive)
- [ ] Speech bubble: Appears above Captain when talking
- [ ] Character size: Approximately 100x150px
- [ ] Idle animation: Subtle floating/bobbing motion
- [ ] Talking animation: Mouth movement + head gestures

from dataclasses import dataclass
from enum import Enum
from typing import Optional, Callable
import random

class CaptainState(Enum):
    IDLE = "idle"
    TALKING = "talking"
    CELEBRATING = "celebrating"
    ENCOURAGING = "encouraging"
    HINTMode = "hint_mode"

@dataclass
class VoiceLine:
    text: str
    category: str
    priority: int  # Higher = more important, interrupts lower priority

class CaptainCosmos:
    def __init__(self):
        self.state = CaptainState.IDLE
        self.current_line: Optional[VoiceLine] = None
        self.voice_lines = self._load_voice_lines()
        self.on_voice_line_complete: Optional[Callable] = None
        
    def _load_voice_lines(self) -> dict:
        """Load voice lines from data/voice_lines.json"""
        pass
    
    def speak(self, category: str, priority: int = 1) -> str:
        """Select and speak a voice line from category"""
        lines = self.voice_lines.get(category, [])
        if not lines:
            return ""
        
        line = random.choice(lines)
        self.current_line = VoiceLine(text=line, category=category, priority=priority)
        self.state = CaptainState.TALKING
        return line
    
    def on_correct_answer(self):
        self.speak("correct")
        
    def on_incorrect_answer(self):
        self.speak("incorrect")
        
    def on_streak_milestone(self, streak: int):
        self.speak("stimulated")
        
    def on_planet_complete(self):
        self.speak("planet_complete")
        
    def on_hint_request(self):
        self.speak("hint_request")
        
    def on_badge_unlock(self):
        self.speak("badge_unlock")
        
    def start_tutorial(self):
        """Return first tutorial line"""
        return self.voice_lines["tutorial"]["welcome"]
        
    def get_tutorial_line(self, step: str) -> str:
        """Get specific tutorial step line"""
        return self.voice_lines["tutorial"].get(step, "")
        
    def on_tts_complete(self):
        """Callback when TTS finishes speaking"""
        self.state = CaptainState.IDLE
        if self.on_voice_line_complete:
            self.on_voice_line_complete()

## Notes & Considerations

- **MVP Approach**: Use TTS for now, professional voice recording is future enhancement
- **Character Personality**: Friendly, encouraging, never condescending
- **Voice Variety**: Multiple voice lines per category to avoid repetition
- **Performance**: Pre-load TTS audio to avoid delays
- **Accessibility**: Closed captions mandatory for hearing-impaired players
- **Customization**: Voice pitch/speed options for different preferences
- **Future**: Option to replace TTS with professional recording if budget allows

### TTS Options for MVP

1. **System TTS** (Recommended for MVP):
   - macOS: `say` command
   - Windows: PowerShell `Speak-Text`
   - Linux: `espeak`
   - Pros: No external dependencies, works offline
   - Cons: Varies by platform, robotic voice

2. **pyttsx3** (Cross-platform TTS library):
   - Pros: Works offline, consistent across platforms
   - Cons: Requires installation, slightly larger footprint

3. **gTTS** (Google Text-to-Speech):
   - Pros: Natural-sounding voices
   - Cons: Requires internet connection

## Overview

This story implements JSON-based data persistence for saving and loading student progress, with auto-save functionality and corruption handling.

## User Story

As a student, I want my progress to be saved automatically so that I don't lose my work if the game closes unexpectedly.

## Acceptance Criteria

### Functional Requirements

- [ ] Auto-save after each word completion
- [ ] Store progress in `data/progress/{student_id}_progress.json`
- [ ] Load existing data on student profile selection
- [ ] Handle missing files gracefully (create new)
- [ ] Handle corrupt files gracefully (backup + recover what possible)
- [ ] Create backup before each save operation
- [ ] Validate JSON structure on load

### Non-Functional Requirements

- [ ] Performance: Save operation completes in < 500ms
- [ ] Accessibility: N/A (background process)
- [ ] Error handling: Queue saves if disk full, retry later

```python
# src/components/data_store.py
import json
import os
import shutil
from datetime import datetime
from typing import Any, Optional

class DataStore:
    def __init__(self, base_path: str = "data/progress"):
        self.base_path = base_path
        os.makedirs(base_path, exist_ok=True)
        
    def _get_file_path(self, student_id: str) -> str:
        return os.path.join(self.base_path, f"{student_id}_progress.json")
    
    def _create_backup(self, file_path: str) -> Optional[str]:
        """Create timestamped backup, return backup path"""
        if not os.path.exists(file_path):
            return None
        backup_dir = os.path.join(os.path.dirname(file_path), "backup")
        os.makedirs(backup_dir, exist_ok=True)
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        backup_path = os.path.join(backup_dir, f"backup_{timestamp}.json")
        shutil.copy2(file_path, backup_path)
        return backup_path
    
    def save(self, student_id: str, data: dict) -> bool:
        """Save data with backup"""
        file_path = self._get_file_path(student_id)
        try:
            # Create backup first
            self._create_backup(file_path)
            # Save new data
            with open(file_path, 'w') as f:
                json.dump(data, f, indent=2)
            return True
        except Exception as e:
            print(f"Save failed: {e}")
            return False
    
    def load(self, student_id: str) -> Optional[dict]:
        """Load data with validation"""
        file_path = self._get_file_path(student_id)
        if not os.path.exists(file_path):
            return self._create_empty_progress(student_id)
        
        try:
            with open(file_path, 'r') as f:
                data = json.load(f)
            if self._validate_schema(data):
                return data
            else:
                return self._recover_from_corrupt(file_path)
        except json.JSONDecodeError:
            return self._recover_from_corrupt(file_path)
    
    def _validate_schema(self, data: dict) -> bool:
        """Check if data has required fields"""
        required = ['student_id', 'sessions', 'mastered_words']
        return all(field in data for field in required)
    
    def _create_empty_progress(self, student_id: str) -> dict:
        """Create new empty progress structure"""
        return {
            'version': '1.0',
            'student_id': student_id,
            'created_at': datetime.now().isoformat(),
            'sessions': [],
            'mastered_words': [],
            'needs_practice': [],
            'achievements': []
        }
```

### Key Implementation Details

1. **DataStore Class** (`src/components/data_store.py`)
   - Main persistence layer for student progress
   - Auto-creates backup before each save operation
   - Handles corrupt/missing files with graceful recovery
   - Maintains last 10 backups per student
   - Student ID sanitization prevents path traversal attacks

2. **BackupManager Class** (`src/utils/file_utils.py`)
   - Timestamped backups with format: `backup_YMD_HMS_MICROSTAMP_filename.json`
   - Automatic cleanup when exceeding MAX_BACKUPS (10)
   - Supports restore from any backup in history

3. **DataValidator Class** (`src/utils/file_utils.py`)
   - Validates JSON syntax before parsing
   - Validates progress schema structure (version, student_id, sessions, etc.)
   - Provides recovery with default values for missing fields

4. **Integration with SessionTracker**
   - SessionTracker now accepts optional DataStore instance
   - Session auto-saves on completion via `save_session()`
   - Updates mastered_words and needs_practice lists based on session data
   - Falls back to in-memory storage if no DataStore provided

5. **Integration with ProgressTracker**
   - ProgressTracker accepts optional DataStore parameter
   - Passes DataStore to SessionTracker for persistence
   - TYPE_CHECKING used to avoid circular imports

### Performance Validation

- Save operation: < 500ms (actual: ~10-50ms for typical data)
- Performance test with 100 sessions + 100 mastered words: passes under 500ms threshold

### Security Considerations

- Student ID sanitization: only alphanumeric, underscore, and hyphen allowed
- Prevents path traversal attacks via malformed student IDs
- No secrets or credentials stored

### Edge Cases Handled

- Missing progress file: Creates empty progress structure
- Corrupt JSON file: Attempts recovery from backup, falls back to empty progress
- Invalid schema: Adds missing fields with default values
- Disk full: Queues save operations for retry
- Empty sessions queue: flush_pending_sessions() returns 0

### Schema Version

Current version: `1.0`

Fields:
- `version`: Schema version for future migrations
- `student_id`: Unique student identifier
- `created_at`: ISO 8601 timestamp of initial save
- `sessions`: Array of session records
- `mastered_words`: Array of word IDs spelled correctly on first attempt
- `needs_practice`: Array of word IDs with multiple attempts
- `achievements`: Array of unlocked achievement IDs

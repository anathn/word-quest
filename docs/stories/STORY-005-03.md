## Overview

Create satisfying sound effects for game events including correct answer chimes, incorrect response tones, streak bonuses, and planet celebration sounds.

## User Story

As a 3rd grade student, I want to hear encouraging sounds when I play so that the game feels responsive and rewarding.

## Acceptance Criteria

### Functional Requirements

- [ ] Correct answer plays ascending chime (C-E-G major chord)
- [ ] Incorrect answer plays gentle descending tone (no harsh sounds)
- [ ] 3-word streak plays layered celebratory sound with rocket boost SFX
- [ ] 5-word streak plays special planet discovery fanfare
- [ ] Planet completion plays victory fanfare with bloom sound
- [ ] All sounds have adjustable volume
- [ ] Sound effects can be toggled on/off
- [ ] Sounds play with <100ms latency

### Non-Functional Requirements

- [ ] Performance: Sound playback adds <5ms overhead
- [ ] Memory: Audio assets use <20MB RAM
- [ ] Volume: Default 50% volume, range 0-100%
- [ ] Format: Support common audio formats (WAV, OGG)
- [ ] Cross-platform: Sounds work on Windows, macOS, Linux

2. **SFXGenerator** - Procedural sound generation for MVP. Creates synthesized sounds without requiring external audio files.

3. **SFXConfig** - Configuration for sound mappings. Maps game events to specific sound effects with parameters.

4. **AudioSettings** - Player preferences for audio. Stores and loads volume levels and mute states.

5. **SoundEvent** - Enum of all sound事件 types in the game.

## UI/UX Requirements

### Screens Involved

- **Spelling Challenge**: Correct/incorrect feedback sounds
- **Celebration Screen**: Streak and planet completion sounds
- **Planet Transition**: Approach and fly-by sounds
- **Settings Screen**: Audio volume controls and mute toggle

### User Interactions

1. Student answers correctly → Ascending chime plays
2. Student answers incorrectly → Gentle descending tone plays
3. Student hits 3-word streak → Rocket boost sound + celebration
4. Student hits 5-word streak → Special discovery fanfare
5. Student completes planet → Victory fanfare with bloom sound

### Audio Design Notes

- **Correct Answer**: Major chord (C-E-G), ascending, 300ms duration
  - Bright, cheerful, encouraging
  - Volume: 50% by default
  
- **Incorrect Answer**: Descending whole step (G→F), 200ms duration
  - Gentle, non-punitive
  - Volume: 30% by default (quieter than success)
  
- **Streak 3**: Layered sound with rocket whoosh
  - Faster tempo, celebratory
  - Volume: 60% by default
  
- **Streak 5**: Special discovery sound
  - Longer fanfare (500ms)
  - More complex harmony
  - Volume: 70% by default
  
- **Planet Complete**: Victory fanfare with bloom
  - Full major chord progression
  - Volume: 80% by default

### Sound Specifications

| Event | Type | Frequencies | Duration | Volume |
|-------|------|-------------|----------|--------|
| Correct | Chime | 523Hz, 659Hz, 784Hz (C-E-G) | 300ms | 50% |
| Incorrect | Tone | 392Hz → 349Hz (G→F) | 200ms | 30% |
| Streak 3 | Layered | 523Hz, 659Hz, 784Hz + whoosh | 400ms | 60% |
| Streak 5 | Fanfare | C-E-G-C (one octave) | 500ms | 70% |
| Planet Complete | Fanfare | C-E-G-C-E (arpeggio) | 600ms | 80% |

## Notes & Considerations

### Edge Cases
- **No Audio Hardware**: Gracefully handle missing audio drivers
- **Audio Driver Failure**: Fall back silently without crashing
- **Concurrent Sounds**: Implement sound queuing or mixing
- **First Launch**: Default volume at 50%

### Implementation Tips
- Use procedural generation for MVP (no asset files needed)
- Pygame.mixer supports multiple simultaneous channels
- Use numpy for waveform generation if needed
- Keep sounds short (<1 second) for responsiveness

### Performance Optimization
- Pre-generate sounds at startup (don't generate on each play)
- Use Pygame's Sound object for efficient playback
- Limit simultaneous sounds to prevent audio clutter
- Cache generated sound data

### Accessibility
- Provide visual feedback as backup for audio
- Allow individual sound volume adjustment (future)
- Support screen reader compatibility
- Consider sound intensity preferences

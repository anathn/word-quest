## Audio Design Principles

1. **Positive Reinforcement**: Success sounds use major chords (cheerful)
2. **Non-Punitive**: Failure sounds are gentle descending tones
3. **Escalating Rewards**: Higher milestones have longer, more complex sounds
4. **Musically Coherent**: All sounds use scales from C major
5. **Responsive**: Short durations (<600ms) maintain game flow

## Edge Cases Handled

- No audio hardware: Silent fallback, no crashes
- Audio driver failure: Graceful degradation
- Concurrent sounds: Multiple channels supported
- First launch: Default volume at 50%
- Pygame not installed: ImportError caught, silent failure

## Implementation Notes

### Audio Initialization Strategy

Sound manager initializes lazily on first use or when screens are created. This prevents blocking on game startup if pygame mixer fails.

### Thread Safety

SoundManager uses singleton pattern with thread-safe initialization:
```python
_sound_manager_lock = threading.Lock()
```

### Volume Hierarchy

Individual sound volumes are defined in config (0.3 to 0.8), then multiplied by global SFX volume (.default 0.5). This allows both per-sound tuning and master control.

### Fallback Behavior

If pygame mixer initialization fails:
- SoundManager still initializes
- `_audio_available` flag is False
- `play()` returns False silently
- No exceptions thrown

### Testable Without Audio Hardware

All tests mock pygame.mixer, allowing unit tests to run in headless environments.

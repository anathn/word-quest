## Overview

This story implements keyboard input handling and validation for spelling answers, supporting both physical and on-screen keyboards with real-time feedback.

## User Story

As a 3rd grade student, I want to type my spelling answer and see my letters appear so that I can submit my response.

## Acceptance Criteria

### Functional Requirements

- [ ] Case-insensitive comparison (ABC = abc = Abc)
- [ ] Trailing/leading spaces ignored
- [ ] Real-time letter display with animation as student types
- [ ] Submit answer on Enter key or "Submit" button press
- [ ] Backspace/Delete removes last letter
- [ ] Input limited to word length (no extra characters)
- [ ] Only alphabetic characters accepted (A-Z, a-z)

### Non-Functional Requirements

- [ ] Performance: Letter appears on screen within 50ms of key press
- [ ] Accessibility: Full keyboard navigation (no mouse required)
- [ ] Error handling: Invalid characters ignored with gentle visual feedback

### Code Structure Suggestions

```python
# src/utils/validators.py
class InputValidator:
    @staticmethod
    def normalize(text: str) -> str:
        """Normalize input for comparison"""
        return text.strip().lower()
    
    @staticmethod
    def is_valid_character(char: str) -> bool:
        """Check if character is A-Z or a-z"""
        return char.isalpha() and len(char) == 1
    
    @staticmethod
    def validate_answer(student_input: str, correct_answer: str) -> bool:
        """Compare normalized inputs"""
        return InputValidator.normalize(student_input) == InputValidator.normalize(correct_answer)

# src/components/input_handler.py
class InputHandler:
    def __init__(self, max_length: int):
        self.input_text = ""
        self.max_length = max_length
        
    def handle_event(self, event):
        if event.type == pygame.KEYDOWN:
            if event.key == pygame.K_BACKSPACE:
                self.input_text = self.input_text[:-1]
            elif event.key == pygame.K_RETURN:
                return self.submit()
            elif len(self.input_text) < self.max_length:
                char = event.unicode
                if InputValidator.is_valid_character(char):
                    self.input_text += char
        return None
    
    def submit(self):
        return self.input_text
```

## UI/UX Requirements

### Screens Involved

- **Spelling Challenge Screen**: Input area below word display

### User Interactions

1. Student types letter → Letter appears in input area with fade-in animation
2. Student presses backspace → Last letter disappears with fade-out

4. Invalid character pressed → No change, subtle shake animation on input field

### Visual Design Notes

- Typed letters: 60pt, white, centered in input area
- Cursor: Blinking vertical line, 2px wide, white, 1Hz blink rate
- Placeholder: Light gray underscores for empty slots
- Submit button: "Submit Answer" text, green (#4CAF50), min 64px height

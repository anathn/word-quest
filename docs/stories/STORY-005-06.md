## Overview

Create visual reward animation when a planet is completed, featuring planet bloom effect with flowers/crystals growth and sparkle particle effects.

As a 3rd grade student, I want to see a beautiful bloom animation when I complete a planet so that I feel a sense of accomplishment and visual satisfaction.

# src/ui/planet_bloom.py
class BloomState:
    IDLE = "idle"
    STARTING = "starting"
    BLOOMING = "bloom"
    HOLDING = "holding"
    FADE_OUT = "fade_out"

class PlanetBloom:
    def __init__(self, planet_sprite):
        self.planet = planet_sprite
        self.state = BloomState.IDLE
        self.progress = 0.0
        self.bloom_duration = 2.0  # seconds
        self.colors = {
            "base": (244, 67, 54),  # Original planet color
            "bloom": (255, 152, 0),  # Brighter bloom color
            "crystals": (255, 235, 59),  # Crystal/flower color
        }
    
    def start_bloom(self):
        """Trigger bloom animation."""
        self.state = BloomState.STARTING
        self.progress = 0.0
    
    def update(self, delta_time: float):
        """Update bloom state."""
        if self.state == BloomState.STARTING:
            self.state = BloomState.BLOOMING
            self.progress = 0.0
        
        if self.state == BloomState.BLOOMING:
            self.progress += delta_time / self.bloom_duration
            if self.progress >= 1.0:
                self.progress = 1.0
                self.state = BloomState.HOLDING
        
        elif self.state == BloomState.HOLDING:
            self.state = BloomState.FADE_OUT
        
        elif self.state == BloomState.FADE_OUT:
            self.progress -= delta_time / self.bloom_duration
            if self.progress <= 0:
                self.state = BloomState.IDLE
    
    def render(self, screen: pygame.Surface, position: tuple):
        """Render planet with bloom effect."""
        # Calculate color interpolation
        intensity = self._get_intensity()
        self.planet.render(
            screen, position,
            bloom_intensity=intensity,
            crystal_density=intensity
        )
    
    def _get_intensity(self) -> float:
        """Return bloom intensity (0.0-1.0)."""
        if self.state == BloomState.IDLE:
            return 0.0
        elif self.state == BloomState.BLOOMING:
            return self.progress
        elif self.state == BloomState.HOLDING:
            return 1.0
        elif self.state == BloomState.FADE_OUT:
            return self.progress
        return 0.0

# src/ui/celebration_renderer.py
class CelebrationRenderer:
    def __init__(self, planet_bloom, sparkle_effect, rocket):
        self.bloom = planet_bloom
        self.sparkles = sparkle_effect
        self.rocket = rocket
        self.active = False
    
    def start_celebration(self, planet_position: tuple):
        """Begin planet completion celebration."""
        self.active = True
        self.bloom.start_bloom()
        self.sparkles.center = planet_position
    
    def update(self, delta_time: float):
        """Update celebration state."""
        if not self.active:
            return
        
        self.bloom.update(delta_time)
        self.sparkles.update(delta_time)
        
        # Emit sparkles during bloom
        if self.bloom.state in [BloomState.BLOOMING, BloomState.HOLDING]:
            self.sparkles.emit(count=3)
        
        # Animate rocket circling planet
        self.rocket.animate_celebration()
        
        # Check if celebration is complete
        if (self.bloom.state == BloomState.IDLE and 
            not self.sparkles.is_active()):
            self.active = False
    
    def render(self, screen: pygame.Surface):
        """Render entire celebration scene."""
        self.bloom.render(screen, self.planet_position)
        self.sparkles.render(screen)
        self.rocket.render(screen)
    
    def is_active(self) -> bool:
        """Return if celebration is ongoing."""
        return self.active

## Notes & Considerations

### Edge Cases
- **Rapid Completion**: If student completes planets quickly, handle overlapping celebrations
- **Low FPS Hardware**: Reduce particle count if performance drops
- **Animation Skip**: Provide option to skip celebration
- **Accessibility**: Reduce sparkle intensity for sensitivity

### Implementation Tips
- Pre-allocate particle objects (object pooling)
- Use additive blending for sparkle effect (glowing)
- Store bloom state in planet data (persist completed state)
- Sync sparkle emission with bloom intensity (more sparkles at peak)

### Performance Optimization
- Limit max particles to 100
- Use simple circle rendering for sparkles
- Cache particle colors to avoid recreation
- Use surface caching for crystal overlays

### Particle System Details
- **Initial Speed**: 1-3 pixels/frame outward
- **Particle Size**: 2-5 pixels
- **Particle Lifespan**: 500-1000ms
- **Emission Rate**: 3-5 particles per update during bloom
- **Color Variation**: Mix of white, gold, and planet accent color
